# Non-Fiction Project Shape

Use this file only when the project layout is ambiguous or when you need to decide which supporting files matter.

## Preferred folders

- `brief/`: thesis, audience, constraints, outline, commissioning notes
- `research/`: source index, excerpts, interview notes, citations, background reading
- `style/`: voice notes, house style, publication rules, recurring editorial preferences
- `drafts/`: article, essay, chapter, or newsletter drafts

## How to read selectively

- Read the target draft first.
- Read `brief/thesis.md` and `brief/constraints.md` before changing argument or confidence level.
- Read `style/voice.md` and `style/house-style.md` before changing cadence, punctuation, citations, or formatting.
- Read `brief/outline.md` when the user asks for drafting, restructuring, or flow fixes.
- Read `brief/audience.md` when explanatory depth, jargon level, or framing depends on readership.
- Read only the research files that support claims actually made in the draft.

## Equivalent filenames you may treat as matching

- `brief/angle.md`, `brief/argument.md`, or `brief/claim.md` can substitute for `brief/thesis.md`
- `brief/readership.md` can substitute for `brief/audience.md`
- `brief/spec.md` or `brief/commission.md` can substitute for `brief/constraints.md`
- `style/style-guide.md` can substitute for `style/house-style.md`
- `research/sources.md`, `research/bibliography.md`, or `research/source-map.md` can substitute for `research/source-index.md`
- `manuscript/`, `chapters/`, or `pieces/` can substitute for `drafts/`

## Evidence rules

- A claim supported by a source in the project may be sharpened or reordered.
- A claim not supported by the project must not be strengthened.
- If the draft contains a quotation, preserve the quotation exactly unless the user provides a corrected version.
- If the draft cites a number, year, or named entity, do not change it unless the project files justify the change.

## Editorial defaults

- Prefer declarative sentences over scene-setting filler.
- Prefer argument over summary.
- Prefer compression over expansion when both preserve meaning.
- Preserve the author's apparent audience and level of sophistication.
